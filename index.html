O que é uma API?

API (Interface de Programação de Aplicativos) é um conjunto de regras e protocolos que permite que diferentes sistemas se comuniquem entre si. Ela define como solicitações e respostas devem ser estruturadas, permitindo que aplicativos compartilhem dados e funcionalidades.

O que caracteriza uma API como REST?

Uma API é considerada RESTful quando segue os princípios da arquitetura REST (Representational State Transfer). Isso significa que ela é baseada em recursos, usa métodos HTTP para interações e é stateless (não mantém estado entre solicitações).

Conceitos Essenciais

  • Recurso: Um recurso é uma entidade acessível por meio da API, como um usuário ou um produto.
  • Identificador de Recurso: É a parte da URL que identifica exclusivamente um recurso, como um ID de usuário em `/usuarios/123`.
  • Tipo de Dados (Data Type): Refere-se ao formato dos dados trocados, geralmente JSON ou XML.

Importância dos Verbos HTTP

Os verbos HTTP (GET, POST, PUT, DELETE, etc.) são importantes em APIs REST porque definem a ação a ser executada em um recurso. Por exemplo:

  • GET: Recupera dados de um recurso.
  • POST: Cria um novo recurso.
  • PUT: Atualiza um recurso existente.
  • DELETE: Remove um recurso.

Principais Códigos de Resposta HTTP

Alguns códigos de resposta HTTP comuns e seus significados incluem:

  • 200 OK: Requisição bem-sucedida.
  • 201 Created: Recurso criado com sucesso.
  • 400 Bad Request: Solicitação inválida.
  • 404 Not Found: Recurso não encontrado.
  • 500 Internal Server Error: Erro interno no servidor.

Modelo de Maturidade

O modelo de maturidade de Richardson é uma escala que avalia o nível de aderência de uma API REST aos princípios RESTful. Vai de 0 (não RESTful) a 3 (totalmente RESTful).
